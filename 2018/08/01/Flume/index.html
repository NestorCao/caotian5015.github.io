Flume

发表于 2018-08-01 | 分类于 bigdata

简介

Flume是Cloudera提供的一个高可用的，高可靠的，分布式的海量日志采集、聚合和传输的系统，Flume支持在日志系统中定制各类数据发送方，用于收集数据；同时，Flume提供对数据进行简单处理，并写到各种数据接受方（可定制）的能力。

Flume架构

  • agent: 是flume最小的日志采集单元，通常涵盖着source.channel.sink组件

  • rawdata:元数据

  • source:负责读取日志源存储在管道中

  • sink：负责从管道中读取数据到目的地

  • event:是flume中数据传输的最小单元，通常有event header和event body组成。header存储的是一些key-value的一些无序集合键值对，eventbody存储的是rawdata的原始数据。Event在Agent传输是高可靠的，只有当Event被Sink完整消费，Channel才会删除event对象。

    思想

    把数据从数据元中收集过来，在将收集到的数据送到指定的目的地。为了保证输送的过程一定成功，在送到目的地(sink)之前，会先缓存数据(channel),待数据真正到达目的地(sink)后，flume在删除自己缓存的数据。 flume提供了对数据采集单元agment的配置.监控.生命州区的管理。

安装Flume

  • Java Runtime Environment - Java 1.7 or laterapache-flume-1.7.0-bin.tar.gz
  • Memory - Sufficient memory for configurations used by sources, channels or sinks
  • Disk Space - Sufficient disk space for configurations used by channels or sinks
  • Directory Permissions - Read/Write permissions for directories used by agent

Spooling Directory Source、HDFS Sink 、 JDBC Channel

采集/root/spooldir目录下的以log结尾的数据，并将采集的数据写入hdfs的/flume/logs目录下。

Taildir Source 、File Channel 、File Roll Sink

采集dir1、dir2的以log结尾的文件,可以检测实时行数据的追加，支持增量式采集。将采集到的结果输出到本地/root/file_roll目录下

AVRO Source 、Memory Channel、Kafka Sink

采集远程目录下的/root/dir1数据，并且将数据发送给Kafka消息队列。
