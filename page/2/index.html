简介

Netty思想

个人感觉Netty这中网络框架相比入Spring Servlet Mybatis等这些在思想上相对复杂抽象些，但是从开发角度讲，基础步骤虽然很墨迹但是完全不需要怎样更改，作为一个开发来说，只需要根据业务有一个完整的业务框架内的一些相关逻辑然后在代码实现上只需对初始化客户端和初始化服务端这两种步骤进行业务更改就可以了{记得导入netty包}；下面从客户端开始说下锁理解的编程步骤。

简单的客户端

在了解这门框架的时候，参照着官网{netty.io}进行开发开源，你会发现基础的步骤不会想差太多，只需要更改里面一个业务逻辑就可以了；说下我理解的客户端netty编程步骤：

7.同步等待客户端关闭连接

简介

我们所说到的三次握手和四次挥手实质就是对应TCP连接建立的过程和断开的过程。那么为什么连接要建立三次或者说断开要断四次呢？简单说下为什么要三次握手四次挥手！！

三次握手

1.三次握手的主要目的就是为了建立相对可靠的通讯信道，关于通讯，其实就是数据的发送与接收，而三次握手的主要目的也就是为了确保双方{自己和对方}的发送和接收机制正常触发。
第一次握手：Client{客户端}什么都不能确认；而在Server{服务端}确认了对方发送正常；
第二次握手：Client确认了自己发送.接收正常，对方发送.接收正常<客户端已完全确认自己无误>；Server确认了自己接收正常.对方发送正常；
第三次握手：Client确认了完后；Server确认了自己发送接收都正常，对方发送接收也正常。

多线程

发表于 2017-08-27 | 分类于 basics

简介

关于多线程在JAVASE中应该是除了反射相对抽象复杂的篇幅了，来简单说一下对线程的理解，里面也有很多查找来自资料和百度百科
1.什么是线程
线程，有时也被称之为轻量进程，是程序执行流的最小单元，一个标准的线程由线程id{Thread.currentThread().getId()获取}.当前指令指针{PC}.寄存器集合和堆栈组成。另外，线程是进程中的一个实体，是被系统独立调度和分派的基本单位，线程自己不拥有系统资源，只拥有一点儿在运行中必不可少的资源，但它可与同属一个进程的其它线程共享进程所拥有的全部资源。一个线程可以创建和撤消另一个线程，同一进程中的多个线程之间可以并发执行。由于线程之间的相互制约，致使线程在运行中呈现出间断性。线程也有就绪、阻塞和运行三种基本状态。就绪状态是指线程具备运行的所有条件，逻辑上可以运行，在等待处理机；运行状态是指线程占有处理机正在运行；阻塞状态是指线程在等待一个事件（如某个信号量），逻辑上不可执行。每一个程序都至少有一个线程，若程序只有一个线程，那就是程序本身。
什么又是多线程
多线程无非就是这个程序或者说这个进程在运行时产生了不止一个线程。
那么为什么要用到多线程呢？其实最终的目的只有一个，那就是更好的利用CPU资源，因为所有的多线程代码其实都可以用单线程来实现，使用它的目的也就是为了高效性{其实编程的发展趋势也一样子，每一个新兴的框架崛起都是因为其开发高效实用！}
什么又是多线程的并行和串行{觉得先了解概念在深入了解比较透彻}
并行：指的就是多个cpu程序或者说多个机器同时去执行一段处理逻辑
串行：通过cpu调度算法，让用户看上去同时执行，实际上从cpu操作层面不是真正的同时。并发往往在场景中有公用的资源，那么针对这个公用的资源往往产生瓶颈，我们会用TPS或者QPS来反应这个系统的处理能力
我们现代的电脑运行程序可以这样理解：cpu分时间片宏观并行，微观串行！

Quartz

发表于 2017-03-13

简介


quartz是一个由java编写的任务调度库，由OpenSymphony组织开源出来。绝大多数公司都会用到任务调度这个功能(感觉是个公司应该都会用到)， 比如公司需要定期执行任务调度生成报表， 或者比如博客什么的定时更新之类的，都可以靠Quartz来完成。
换言之他的任务场景：就比如定时的消息推送.定时抢购.定时发送邮件.定时发送博客.定时统计。（反正于定时相关就找他就行）

任务调度：现在有N个任务（程序），要求在指定时间执行，比如每周二3点执行任务A、每天相隔5s执行任务B等等，这种多任务拥有多种执行策略就是任务调度。
而quartz的核心作用，是使任务调度变得丰富、高效、安全，开发者只需要调几个quartz接口并做简单配置，即可实现上述需求。
quartz号称能够同时对上万个任务进行调度，拥有丰富的功能特性，包括任务调度、任务持久化、可集群化、插件等
